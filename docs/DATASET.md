# Dataset Documentation

## Overview

This document describes the structure, format, and usage of the ground truth dataset generated by the Devanagari OCR annotation platform.

> For agent constraints and execution logs, see [`AGENTS.md`](../AGENTS.md) in the project root.

## Dataset Structure

### SQLite Database Schema

The primary data store is an SQLite database (`data/annotations.db`) with the following schema:

#### Table: `page_annotations`

| Column | Type | Description |
|--------|------|-------------|
| `page_id` | TEXT PRIMARY KEY | Unique identifier (e.g., "doc_A_page_001") |
| `pdf_source` | TEXT | Path to source PDF file |
| `page_number` | INTEGER | Page number within the PDF |
| `image_path` | TEXT | Path to high-resolution PNG image |
| `ocr_draft` | TEXT | Initial OCR output from Gemini |
| `ground_truth` | TEXT | Human-corrected transcription |
| `status` | TEXT | Status: 'pending', 'completed', 'flagged' |
| `wer` | REAL | Word Error Rate (0.0 to 1.0+) |
| `cer` | REAL | Character Error Rate (0.0 to 1.0+) |
| `annotation_time_sec` | REAL | Time spent correcting (seconds) |
| `created_at` | TIMESTAMP | Record creation time |
| `updated_at` | TIMESTAMP | Last update time |

## Data Export Formats

### 1. CSV Export

Standard format for data analysis and sharing:

```python
import pandas as pd

df = pd.read_sql_query("SELECT * FROM page_annotations WHERE status='completed'", 
                       sqlite3.connect("data/annotations.db"))
df.to_csv("ground_truth_export.csv", index=False, encoding='utf-8')
```

**CSV Schema:**
- All columns from database
- UTF-8 encoding with BOM for Excel compatibility
- Unix line endings (\n)

### 2. JSONL Export

Format for machine learning training:

```json
{"page_id": "doc_A_page_001", "image_path": "data/images/doc_A_page_001.png", "text": "Ground truth text here...", "metadata": {"source": "doc_A.pdf", "page": 1}}
```

### 3. Hugging Face Datasets Format

Ready for upload to Hugging Face Hub:

```python
from datasets import Dataset
import sqlite3

conn = sqlite3.connect("data/annotations.db")
df = pd.read_sql_query("SELECT * FROM page_annotations WHERE status='completed'", conn)

```

## Annotation Guidelines

### Ground Truth Standards

1. **Character Accuracy**
   - Transcribe exact Unicode characters
   - Preserve all ligatures and conjuncts
   - Maintain original spacing

2. **Formatting Rules**
   - Preserve line breaks for verse/prose structure
   - Keep paragraph breaks
   - Retain indentation if meaningful

3. **Special Cases**
   - Use `[unreadable]` for illegible text
   - Use `[image]` for non-text elements (diagrams, etc.)
   - Use `[page_break]` for explicit page boundaries

4. **Quality Thresholds**
   - Target CER < 0.05 (5%) for publication
   - Target WER < 0.10 (10%) for publication
   - Flag documents with error rates >20% for re-scan

### Annotation Workflow

1. **Preparation**
   - Upload PDFs to `data/raw/`
   - Run batch conversion script
   - Wait for automatic OCR processing

2. **Correction**
   - Load image in web interface
   - Review OCR draft
   - Use transliteration tool for efficient typing
   - Save corrections

3. **Review**
   - Check WER/CER metrics
   - Flag problematic pages
   - Verify formatting

4. **Export**
   - Generate CSV/JSONL
   - Validate character encoding
   - Archive completed dataset

## Dataset Statistics

### Query Examples

**Total annotated pages:**
```sql
SELECT COUNT(*) FROM page_annotations WHERE status = 'completed';
```

**Average error rates:**
```sql
SELECT AVG(wer) as avg_wer, AVG(cer) as avg_cer 
FROM page_annotations 
WHERE status = 'completed';
```

**Annotation productivity:**
```sql
SELECT SUM(annotation_time_sec)/3600 as total_hours,
       COUNT(*) as total_pages,
       SUM(annotation_time_sec)/COUNT(*) as seconds_per_page
FROM page_annotations 
WHERE status = 'completed';
```

**Pages by source:**
```sql
SELECT pdf_source, COUNT(*) as page_count, AVG(cer) as avg_cer
FROM page_annotations 
WHERE status = 'completed'
GROUP BY pdf_source;
```

## Data Quality Assurance

### Automated Checks

1. **Character Validation**
   - Verify all characters in Devanagari Unicode range (U+0900-U+097F)
   - Flag suspicious characters or mixed scripts
   - Validate UTF-8 encoding

2. **Length Consistency**
   - Compare ground truth length to image dimensions
   - Flag extremely short outputs (<10 chars for full page)
   - Flag outputs longer than expected

3. **Duplicate Detection**
   - Check for duplicate page_ids
   - Identify potential double-annotations

### Manual Review Triggers

Pages automatically flagged for manual review:
- CER > 0.20 (20% errors)
- WER > 0.30 (30% errors)
- Annotation time > 300 seconds (5 minutes)
- Status manually set to 'flagged'

## Versioning and Backup

### Backup Strategy

1. **Database Backup**
   - Hourly snapshots during active annotation
   - Daily full backups
   - Weekly archive to cloud storage

2. **Export Archives**
   - Versioned dataset exports
   - Immutable after release
   - Checksum verification

```python

# Error rate distribution
df['cer'].hist(bins=50)
plt.xlabel('Character Error Rate')
plt.ylabel('Frequency')
```
